Version Control System Source Control Management/ Revision Control

Version control systems work to give users better access to past versions of files. Previously, users would have to save multiple file with varying names and dates, creating more room for inconsistencies and error. It also disables multiple users from working on the same files at once. VCS uses one main repository where a single user, and multiple users, can check out files, edit them, and "commit" them back into the repository. Benefits of this technology include knowledge of: who edited the file, when they changed it, what they changed, and ability to merge the changes to the same file. This enables a virtual community that can work effieciently together on the same project using separate devices, without losing previous edits. Amazing!

Remote Repository Multiple GIT Users

In short, a remote repostitory is a repostiory that accesed on a remote server, via the internet, and shared amongst authorized team members. It is understood as versions of a project hosted on the internet or alternate network. This is the common ground between users working on the same project, as it allows users to push, pull, and fetch information from each of the repositories shared. It is critical for users to understand how to push (git add), or put information into the remote server for team to access, and pull(git fetch), or add information into your repository for inspection, and ability to merge information.

Local Repository Single GIT User

A local repository is a repository that is located on the local machine of a user. This is the users perosonal copy of the repository for the user to edit aspects about the code. To create a local repository, one option is to use Git init to initialize the repository. Once done editing, the user can add the code to the staging area, using Git add, then commit the changes to the repository, using Git commit.After committed, the user can push the changes made in the local repository to the remote repository utilizing Git commit. The other method involves cloning a repository, using Git clone, and follow the same steps as if they used Git init.

What is GIT Understanding the Best VCS

GIT was created in 2005 by innovator Linus Torvalds, when he wanted to create an environment where single and multiple content creators are able to update files simultaneously, and effienciently. To do this, a Distributed Version Control System is used. This allows for remote repository to house all information, while users operating on a local repository can update the remote repository on their device. Since every clone, or branch, of the repository is saved on each local device, it nullifies the ability to loose all data, because it can be restored by one of the copies. Working in one branch will not show up in the remote repository unitl it is merged into other branches and commited to remote repository. Benefits of GIT include a complete copy of the remote repository stored on the local repository and, the use of a local repository enabling the user to edit content offline.

Basic GIT Commands How to use GIT

GIT has numerous commands that enable the user many tools to work on their project more effieciently. Here is a list of basic commands along with their funtion.

  • GIT config -- global user.name and GIT config -- global user.email are used to configure the name and email address associated with commits made to the remote repository.
  • GIT init is used to create a new local repository
  • GIT add is used to add one or more files to the staging area in GIT
  • GIT commit is used to commit the changes made to the file to the head, or current branch. This does not save the changes to the remote repository yet
  • GIT push is used for sending the changes made in the master branch to the remote repository
  • GIT checkout is used to toggle between each branch in the local repository
  • GIT pull is used to fetch and merge all changes in the remote repository into the local repository
  • GIT merge is used to merge a seperate branch into the current branch used.
